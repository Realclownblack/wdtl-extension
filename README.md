# Wdtl — VS Code Extension

VS Code extension for the **Wdtl** data pipeline language (`.w`). Includes:

- Full **syntax highlighting** (all commands, functions, types, operators, and literals from the manual).
- **IntelliSense**: autocomplete for keywords/functions/types with documentation, hover (mousing
  over shows the syntax and description), and parameter help when opening `(` in functions like
  `csv(...)`, `sum(...)`, `round(...)`, etc.
- **LSP Client**: connects to the native `w lsp` language server for diagnostics, go-to-definition,
  find references, and any other capabilities your server exposes — all without extra configuration.
- Ready-made **snippets** for all commands (`load-csv`, `filter`, `groupby`, `join`, `pivot`,
  `let-if`, `export-csv`, etc. — type the prefix and press Tab).
- Custom **file icon** (your "W" logo) for `.w` files in the Explorer.

```
wdtl-extension/
├── package.json                  # extension manifest
├── language-configuration.json   # comments (#), brackets, auto-closing
├── tsconfig.json                 # TypeScript compiler config
├── src/
│   ├── extension.ts              # activates IntelliSense + LSP client
│   └── wLanguageData.ts          # database with all commands/functions/types
├── out/                          # compiled JS (generated by tsc)
├── syntaxes/w.tmLanguage.json    # TextMate grammar (highlight)
├── snippets/w.json               # snippets for all commands
├── fileicons/                    # icon theme with your logo for .w
│   ├── w-icon-theme.json
│   ├── w-icon.png                # your uploaded icon
│   └── default-*.svg             # neutral fallback icons (generic file/folder)
├── examples/exemplo.w            # test file
└── icon.png                      # the extension's own icon (Marketplace/Extensions view)
```

## Prerequisites

- **[Node.js](https://nodejs.org)** (v18 or newer) — required to compile the extension.
- **`w` binary in your `PATH`** — the extension calls `w lsp` to start the language server.
  If the binary is not on `PATH`, the LSP client will show an error in the Output panel
  (the rest of IntelliSense / highlighting / snippets still works fine).

## How to install (without using the Marketplace)

There are three ways. **Option A** is the fastest (pre-built). **Option B** generates an
installable `.vsix` file, recommended if you want to share the extension with other people.

### Option A — Pre-built zip (fastest)

1. Unzip the `.zip` you downloaded.
2. Inside the folder run:

```bash
npm install          # installs vscode-languageclient
npm run compile      # compiles TypeScript → out/
```

3. Rename (or keep) the folder as `wdtl-extension`.
4. Copy the entire folder to VS Code's extensions directory:
   - **Windows**: `%USERPROFILE%\.vscode\extensions\`
   - **macOS / Linux**: `~/.vscode/extensions/`
5. Close and reopen VS Code (or run **Developer: Reload Window** from the command palette,
   `Ctrl+Shift+P`).
6. Open any `.w` file (for example `examples/exemplo.w`) — highlighting, IntelliSense and
   LSP should already be working.

### Option B — Generate a `.vsix` with `vsce` (installable via "Install from VSIX")

```bash
cd wdtl-extension
npm install
npm run compile
npm install -g @vscode/vsce
vsce package

npx @vscode/vsce package
```

This generates a `wdtl-extension.vsix` file. In VS Code:

1. Open the command palette (`Ctrl+Shift+P`) → **Extensions: Install from VSIX...**
2. Select the generated `.vsix` file.
3. Reload the window when prompted.

### Option C — Test without installing (developer mode, F5)

1. Open the `wdtl-extension` folder in VS Code.
2. Run `npm install && npm run compile` in the terminal.
3. Press `F5` (or run **Run and Debug → Run W Language extension**).
4. A new VS Code window opens with the extension already active.

## LSP Client

The extension automatically starts `w lsp` as a language server process using stdio transport
when you open a `.w` file. The client is configured in `src/extension.ts`:

```typescript
const serverOptions: ServerOptions = {
  command: 'w',
  args: ['lsp'],
  transport: TransportKind.stdio,
};

const client = new LanguageClient('w-lsp', 'W Language Server', serverOptions, clientOptions);
client.start();
```

Any LSP feature your server announces (diagnostics, go-to-definition, find references,
code actions, formatting, etc.) will automatically appear in VS Code without extra extension code.

To see the server output: **View → Output → W Language Server**.

## Activating the file icon

The custom icon comes inside a separate **icon theme** (VS Code's standard way of adding file
icons). To activate it:

1. `Ctrl+Shift+P` → **Preferences: File Icon Theme**
2. Choose **W Language Icons**.

> **Note:** this icon theme is minimalist — it only overrides the icon for `.w` files.
> If you use another full icon theme (Material Icon Theme, vscode-icons, etc.), you can keep it —
> the `w` icon will only appear after switching to "W Language Icons".

## Testing IntelliSense

Open `examples/exemplo.w` and try:

- Type `load-` on an empty line → snippets like `load-csv`, `load-excel`, `load-json`, etc. appear.
- Type `gro` → suggestion for `GROUPBY` with the full syntax.
- Hover over `sum`, `coalesce`, `JOIN`, `Float64`, etc.
- Inside `round(` → parameter help appears (`round(number, decimals)`).

## Customizing

- **Add/edit IntelliSense commands**: edit `src/wLanguageData.ts` (each entry has `label`, `detail`,
  and `doc`). Run `npm run compile` after editing.
- **Add/edit snippets**: edit `snippets/w.json`.
- **Adjust highlight colors**: the grammar in `syntaxes/w.tmLanguage.json` uses standard TextMate
  scopes (`keyword.control`, `support.function`, `storage.type`, etc.), so it automatically
  respects whichever color theme (Dark+, Light+, etc.) you're already using in VS Code.
- **Change the icon**: replace `fileicons/w-icon.png` with another PNG/SVG (keep the same name or
  update the path in `fileicons/w-icon-theme.json`).
- **LSP server command**: if your binary is named differently or not on `PATH`, edit the
  `serverOptions` in `src/extension.ts` and recompile.
